About the Project

This project studies how well current AI translation and NLP approaches handle African languages when localizing real software interfaces (Voyant Tools), and how human validation can preserve meaning and nuance.

The problem

  • Monolingual records hide African terms
  • Eurocentric metadata flattens nuance
  • Epistemological representation and understanding of Textual Analysis Concepts

The solution

  • Parallel metadata across African languages
  • Human-in-the-loop validation & provenance
  • Open datasets + pluggable models (Ollama/OpenAI)

Future work

  • Expand language coverage and domain glossaries
  • ☐ Add semantic similarity checks (embeddings)
  • ☐ Deeper ontology links without flattening nuance

Workflow (high level)

  • Source strings are imported from Voyant’s CSV into a database.
  • AI drafts are generated (e.g., NLLB-200; optionally LLMs) while protecting placeholders and HTML tags.
  • Verification uses back-translation and semantic similarity scoring (XLM-R-based) to detect divergences.
  • QA checks detect placeholder/tag mismatches and other formatting risks.
  • Human review approves or rejects translations (human-in-the-loop).

Why this matters

Even when full NLP analysis for a language is not yet available inside Voyant, multilingual interface “skins” are a meaningful step toward accessibility and language justice in digital scholarship.
